Aplicaci√≥n de la segmentaci√≥n#

¬øQu√© es la segmentaci√≥n?#

En los cuadernos anteriores, solo hemos abordado problemas de clasificaci√≥n, es decir, predecir una clase a partir de una imagen (por ejemplo, ‚Äúesta es una imagen del n√∫mero 5‚Äù o ‚Äúesta es una imagen de un gato‚Äù). Sin embargo, existen otras dos categor√≠as de an√°lisis de im√°genes: la detecci√≥n y la segmentaci√≥n.

Detecci√≥n de objetos en una imagen#

La detecci√≥n de objetos en una imagen consiste en dibujar un recuadro alrededor del objeto de inter√©s. Por ejemplo, si queremos detectar gatos y la imagen contiene un perro y un gato, entrenamos un modelo para dibujar un cuadro alrededor del gato.

A continuaci√≥n, se muestra una imagen para visualizar la idea de la detecci√≥n de objetos:

Imagen extra√≠da del art√≠culo.

Como pueden imaginar, esta es una tarea muy interesante en el campo del procesamiento de im√°genes, pero no es lo que abordaremos en este cuaderno.

Segmentaci√≥n de objetos en una imagen#

La segmentaci√≥n tiene como objetivo determinar la clase a la que pertenece cada p√≠xel de la imagen (por ejemplo, el p√≠xel en la posici√≥n \((120, 300)\) pertenece a un perro). El objetivo es obtener informaci√≥n muy precisa sobre el contenido de la imagen a nivel de p√≠xel.

Imagen extra√≠da del art√≠culo.

Existen varios tipos de segmentaci√≥n:

  • Segmentaci√≥n sem√°ntica: Consiste en clasificar cada p√≠xel de la imagen independientemente de la instancia (por ejemplo, todos los coches se colorear√°n de rojo). Este es el tipo de segmentaci√≥n que se muestra en la imagen anterior.

  • Segmentaci√≥n de instancias: Aqu√≠, cada instancia de una misma clase se segmenta con un color diferente (por ejemplo, el primer coche en rojo, el segundo en naranja, etc.).

  • Tambi√©n existen otros tipos de segmentaci√≥n. Para m√°s informaci√≥n, pueden consultar este art√≠culo.

En este cuaderno, realizaremos un ejemplo de segmentaci√≥n sem√°ntica.

Conjunto de datos utilizado#

Utilizamos el conjunto de datos Oxford-IIIT Pet Dataset, que contiene un total de \(7349\) im√°genes, de las cuales \(4978\) son de perros y \(2371\) de gatos. Para cada imagen, se incluyen anotaciones de segmentaci√≥n, as√≠ como un recuadro alrededor de la cabeza del animal.

Solo utilizaremos la anotaci√≥n correspondiente a la segmentaci√≥n del animal.

Para obtener y utilizar este conjunto de datos, es necesario descargarlo directamente desde el sitio web, ya que la versi√≥n proporcionada por torchvision no incluye las m√°scaras de segmentaci√≥n.

Cuando el conjunto de datos que deseas utilizar no est√° disponible directamente en torchvision, es necesario crear una clase que herede de la clase Dataset para especificar c√≥mo gestionar los datos de dicho conjunto.

# Import n√©cessaire pour le projet
from PIL import Image
import os
import matplotlib.pyplot as plt
from torch.utils.data import Dataset
import torchvision.transforms as T
import torch.nn as nn
from torch.utils.data import DataLoader
import torch
import numpy as np

class datasetSeg(Dataset):
  def __init__(self, path):
    self.path = path
    self.imagesPath,self.masksPath=self.loadDataset()
    
    self.resize=T.Resize((100, 100)) # Valeur de resize pour les images
    self.toTensor=T.ToTensor()
  
  def __len__(self):
    return len(self.imagesPath)
  
  def loadDataset(self): # R√©cuperation des liens vers les images et les annotations
    images=os.listdir(self.path+"images/")
    images=sorted(images)
    masks=os.listdir(self.path+"annotations/trimaps/")
    masks=sorted(masks)
    masks=[ mask for mask in masks if "._" not in mask ]
    images=[image for image in images if ".mat" not in image]
    return images, masks
  
  def __getitem__(self, index):
    image=Image.open(self.path+"images/"+self.imagesPath[index]).convert('RGB')
    # Resize des images et r√©cuperation du masque de segmentation
    trimap=self.resize(Image.open(self.path+"annotations/trimaps/"+self.masksPath[index]).convert('L'))
    trimap=np.array(trimap)
    # Cr√©ation des masques pour les classes
    class1 = (trimap == 1).astype(np.uint8)  
    class2 = (trimap == 2).astype(np.uint8)  
    class3 = (trimap == 3).astype(np.uint8)
    mask = np.stack([class1, class2, class3], axis=0)
    return self.toTensor(self.resize(image)), torch.tensor(mask)

Arquitectura del modelo: U-Net#

A diferencia de un problema de clasificaci√≥n de im√°genes, la salida de nuestra red neuronal debe tener la misma dimensi√≥n que la entrada en t√©rminos de resoluci√≥n y un canal por cada clase que deseamos segmentar (por ejemplo, si queremos segmentar diez clases en im√°genes de entrada de tama√±o \(224 \times 224 \times 3\), la salida ser√° de tama√±o \(224 \times 224 \times 10\)). Una arquitectura cl√°sica de red convolucional, donde la resoluci√≥n disminuye progresivamente hasta llegar a una capa totalmente conectada de clasificaci√≥n, no es adecuada para este caso. En su lugar, podemos utilizar la arquitectura U-Net.

A continuaci√≥n, se muestra c√≥mo es la arquitectura U-Net:

Figura extra√≠da de este art√≠culo.

Como pueden observar, la arquitectura es bastante at√≠pica. Consiste en una ruta de contracci√≥n (llamada codificador), que reduce el tama√±o de la imagen como en una CNN cl√°sica, y una ruta de expansi√≥n (llamada decodificador), que aumenta el tama√±o de la imagen hasta alcanzar una resoluci√≥n id√©ntica a la entrada. Las conexiones indicadas por las flechas negras permiten conservar informaci√≥n local precisa, mientras que la parte codificador/decodificador permite detectar relaciones m√°s abstractas entre diferentes p√≠xeles.

Esta arquitectura fue introducida por primera vez para la segmentaci√≥n de im√°genes m√©dicas en este art√≠culo. Actualmente, se utiliza en numerosos campos (segmentaci√≥n, reducci√≥n de ruido, difusi√≥n, etc.).

Acerca de la convoluci√≥n transpuesta#

Hemos visto en detalle las capas de convoluci√≥n, que permiten realizar operaciones en im√°genes con par√°metros entrenables. Sabemos que, gracias al par√°metro stride, las capas de convoluci√≥n permiten conservar la resoluci√≥n inicial o reducirla. Sin embargo, a veces es necesario aumentar la resoluci√≥n de la imagen (por ejemplo, para la parte del decodificador de la U-Net descrita anteriormente).

Para ello, podemos considerar varias opciones:

  • Interpolaci√≥n seguida de una convoluci√≥n: Similar al pooling, pero en sentido inverso, esta idea consiste en utilizar una funci√≥n de interpolaci√≥n no entrenable para ampliar artificialmente la imagen.

  • Convoluci√≥n transpuesta: Esta operaci√≥n act√∫a como la inversa de una convoluci√≥n y permite aumentar el tama√±o de la imagen. A continuaci√≥n, se muestra un ejemplo de convoluci√≥n transpuesta con un kernel de tama√±o \(2 \times 2\). Es una operaci√≥n que contiene par√°metros entrenables.

Figura extra√≠da de este art√≠culo.

Para quienes deseen profundizar y conocer las diferencias principales entre ambas t√©cnicas, recomiendo leer este art√≠culo.

En nuestra implementaci√≥n, utilizaremos la convoluci√≥n transpuesta.

Implementaci√≥n en PyTorch#

Es hora de implementar la arquitectura U-Net en PyTorch:

# Fonction pour combiner couche de convolution, activation ReLU et BatchNorm pour √©viter les copier coller 
def conv_relu_bn(input_channels, output_channels, kernel_size, stride, padding):
  return nn.Sequential(
    nn.Conv2d(input_channels, output_channels, kernel_size, stride, padding),
    nn.ReLU(),
    nn.BatchNorm2d(output_channels,momentum=0.01)
  )
    
# Idem mais avec la convolution transpos√©e
def convT_relu_bn(input_channels, output_channels, kernel_size, stride, padding):
  return nn.Sequential(
    nn.ConvTranspose2d(input_channels, output_channels, kernel_size, stride, padding),
    nn.ReLU(),
    nn.BatchNorm2d(output_channels,momentum=0.01)
  )

class UNet(nn.Module):
  def __init__(self, num_classes=3):
    super().__init__()
    # encoder
    self.conv1=conv_relu_bn(3, 64, 3, 1, 1)
    self.conv2=conv_relu_bn(64, 64, 3, 1, 1)
    self.maxPool1=nn.MaxPool2d(2, 2)
    
    self.conv3=conv_relu_bn(64, 128, 3, 1, 1)
    self.conv4=conv_relu_bn(128, 128, 3, 1, 1)
    self.maxPool2=nn.MaxPool2d(2, 2)
    
    #Module central
    self.conv5=conv_relu_bn(128, 256, 3, 1, 1)
    self.conv6=conv_relu_bn(256, 256, 3, 1, 1)
    
    # D√©codeur
    self.convT1=convT_relu_bn(256, 128, 4, 2, 1)
    self.conv7=conv_relu_bn(256, 128, 3, 1, 1)
    
    self.convT2=convT_relu_bn(128, 64, 4, 2, 1)
    self.conv8=conv_relu_bn(128, 64, 3, 1, 1)
    
    # On va pr√©dire un channel par classe
    self.conv9=conv_relu_bn(64, num_classes, 3, 1, 1)     
    self.sigmoid=nn.Sigmoid()
  
  def forward(self,x):
    # Encodeur
    x=self.conv1(x)
    x1=self.conv2(x)
    
    x=self.maxPool1(x1)
    
    x=self.conv3(x)
    x2=self.conv4(x)
    x=self.maxPool2(x2)
    
    # Module central
    x=self.conv5(x)
    x=self.conv6(x)

    #D√©codeur
    x=self.convT1(x)
    x=torch.cat((x,x2),dim=1)
    x=self.conv7(x)
    
    x=self.convT2(x)
    x=torch.cat((x,x1),dim=1)
    x=self.conv8(x)
    
    x=self.conv9(x)
    x=self.sigmoid(x)
    return x

Entrenamiento del modelo#

Para entrenar el modelo, no distinguiremos entre perros y gatos, sino que buscaremos simplemente identificar los p√≠xeles que pertenecen al animal, los p√≠xeles del borde y los p√≠xeles del fondo.

# Nous d√©fissons
model=UNet(num_classes=3).to('cuda')
criterion=nn.CrossEntropyLoss()
lr=0.001
optimizer=torch.optim.Adam(model.parameters(), lr=lr)
epochs=5

La base de datos es bastante extensa y el modelo es bastante profundo. Es posible que el entrenamiento dure varias decenas de minutos si su GPU no es muy potente. (Si no tiene GPU, no recomiendo intentar entrenar el modelo).

for epoch in range(epochs):
  train_loss=0
  for images,masks in train_dataloader:
    images=images.to('cuda')
    masks=masks.to('cuda').float()
    optimizer.zero_grad()
    seg=model(images)
    loss=criterion(seg,masks)
    loss.backward()
    optimizer.step()
    train_loss += loss.item()

  val_loss=0
  for images,masks in val_dataloader:
    images=images.to('cuda')

      seg=model(images)
      loss=criterion(seg,masks)
    val_loss += loss.item() 
  print(f"step {epoch} train loss {val_loss/len(val_dataloader)}")

Ahora podemos calcular la precisi√≥n en nuestros datos de prueba.

def calculate_class_accuracy(preds, masks, class_idx):
  #On convertit les pr√©dictions en valeur entre 0 et 1 pour chaque classe
  preds = torch.argmax(preds, dim=1)
  
  # On ne r√©cup√®re que les pixels de la classe d'int√™ret
  preds_class = (preds == class_idx).float()
  masks_class = (masks == class_idx).float()
  
  # Calculer la pr√©cision pour la classe choisie
  correct = (preds_class == masks_class).float()
  accuracy = correct.sum() / correct.numel()
  return accuracy

model.eval()
test_accuracy = 0.0
num_batches = 0


with torch.no_grad(): # D√©sactivation du calcul du gradient
  for images, masks in test_dataloader:
    images = images.to('cuda')
    masks = masks.to('cuda').long()
    
    seg = model(images)
    
    # On calcule la pr√©cision pour la classe d'int√™ret (0 correspond √† la segmentation de l'animal)
    class_idx = 0
    batch_accuracy = calculate_class_accuracy(seg, masks, class_idx)
    
    test_accuracy += batch_accuracy.item()
    num_batches += 1

# On calcule la pr√©cision moyenne sur l'ensemble du dataset de test
test_accuracy /= num_batches
print(f'Pr√©cision pour la segmentation de l animal : {test_accuracy*100:.1f}%')
