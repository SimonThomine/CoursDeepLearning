En este curso, construiremos nuestra primera red neuronal utilizando la biblioteca micrograd. Micrograd es una biblioteca sencilla y f√°cil de entender, especializada en el c√°lculo autom√°tico de gradientes. Para dominarla mejor, puedes ver el v√≠deo de introducci√≥n de Andrej Karpathy (en ingl√©s). Este cuaderno tambi√©n se inspira en el cuaderno proporcionado en el repositorio de micrograd.

Descenso de gradiente estoc√°stico (SGD)#

Antes de continuar, hablemos sobre el descenso de gradiente estoc√°stico (SGD). Para aplicar el algoritmo de descenso de gradiente en un conjunto de datos de tama√±o \(N\), en teor√≠a ser√≠a necesario calcular la p√©rdida y el gradiente en cada elemento antes de actualizar los pesos. Este m√©todo garantiza una disminuci√≥n de la p√©rdida en cada iteraci√≥n, pero es muy costoso para conjuntos de datos donde \(N\) es grande (a menudo \(N > 10^6\)). Adem√°s, ser√≠a necesario almacenar los gradientes de todos los \(N\) elementos en memoria, lo cual es imposible para grandes conjuntos de datos.

Para resolver este problema, utilizamos mini-batches, que son grupos de muestras del conjunto de datos. La optimizaci√≥n se realiza como en el descenso de gradiente cl√°sico, pero la actualizaci√≥n de los pesos se realiza en cada mini-batch (por lo tanto, con mayor frecuencia). Esto hace que el proceso de optimizaci√≥n sea m√°s r√°pido y permite manejar grandes cantidades de datos. El tama√±o de un mini-batch se denomina batch size y su valor suele ser \(16\), \(32\) o \(64\). Para obtener m√°s informaci√≥n sobre el descenso de gradiente estoc√°stico, consulta Wikipedia o este art√≠culo.

Definamos en Python una funci√≥n para recuperar batch_size elementos aleatorios de nuestro conjunto de datos.

def get_batch(batch_size=64):
  ri = np.random.permutation(X.shape[0])[:batch_size]
  Xb, labels = X[ri], y[ri]
  #inputs = [list(map(Value, xrow)) for xrow in Xb] # OLD
  # Conversion des inputs en Value pour pouvoir utiliser micrograd
  inputs = [list([Value(xrow[0]),Value(xrow[1])]) for xrow in Xb]
  return inputs,labels

Funci√≥n de p√©rdida#

Para entrenar nuestra red neuronal, debemos definir una funci√≥n de p√©rdida (loss). En nuestro caso, tenemos dos clases y queremos maximizar el margen entre los ejemplos que pertenecen a clases diferentes. A diferencia de la p√©rdida de log-verosimilitud negativa utilizada previamente, buscamos maximizar esta diferencia, lo que hace que nuestro m√©todo sea m√°s robusto ante nuevos elementos.

Utilizamos la p√©rdida de margen m√°ximo (max-margin loss), definida por: \(\text{loss} = \max(0, 1 - y_i \cdot \text{score}_i)\)

def loss_function(scores,labels):
  # La fonction .relu() prend le maximum entre 0 et la valeur de 1 - yi*scorei
  losses = [(1 - yi*scorei).relu() for yi, scorei in zip(labels, scores)]
  # On divise le loss par le nombre d'√©l√©ments du mini-batch
  data_loss = sum(losses) * (1.0 / len(losses))
  return data_loss

Entrenamiento del modelo#

Ahora que tenemos los elementos clave del entrenamiento, es hora de definir nuestro bucle de entrenamiento.

# D√©finissons nos hyper-param√®tres d'entra√Ænement 
batch_size=128
iteration=50

Ahora podemos iniciar el entrenamiento de nuestro modelo:

for k in range(iteration):
    
  # On r√©cup√®re notre mini-batch random
  inputs,labels=get_batch(batch_size=batch_size)

  # On fait appel au mod√®le pour calculer les scores Y
  scores = list(map(model, inputs))
  
  # On calcule le loss
  loss=loss_function(scores,labels)


  accuracy = [(label > 0) == (scorei.data > 0) for label, scorei in zip(labels, scores)]
  accuracy=sum(accuracy) / len(accuracy)
  
  # Remise √† z√©ro de valeurs de gradients avant de les calculer
  model.zero_grad() 
  # Calcul des gradients gr√¢ce √† l'autograd de micrograd
  loss.backward() 
  
  # Mise √† jour des poids avec les gradients calcul√©s (SGD)
  learning_rate = 1.0 - 0.9*k/100 # On diminue le learning rate au fur et √† mesure de l'entra√Ænement
  for p in model.parameters():
      p.data -= learning_rate * p.grad
  
  if k % 1 == 0:
      print(f"step {k} loss {loss.data}, accuracy {accuracy*100}%")

Como pueden observar, la p√©rdida (loss) no disminuye sistem√°ticamente en cada paso del entrenamiento. Esto se explica por el descenso de gradiente estoc√°stico: el hecho de no considerar todo el conjunto de datos en cada iteraci√≥n introduce un componente aleatorio.
