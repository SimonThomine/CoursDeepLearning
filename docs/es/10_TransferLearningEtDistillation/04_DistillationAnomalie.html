Destilaci√≥n de conocimientos para la detecci√≥n no supervisada de anomal√≠as#

Este cuaderno implementa la destilaci√≥n de conocimientos para la detecci√≥n no supervisada de anomal√≠as.

Nos basamos en el art√≠culo Student-Teacher Feature Pyramid Matching for Anomaly Detection y su c√≥digo asociado. La siguiente figura, extra√≠da del art√≠culo, muestra el funcionamiento del m√©todo:

Selecci√≥n del backbone y del conjunto de datos#

Backbone y funci√≥n de p√©rdida#

El art√≠culo utiliza ResNet18 como arquitectura de red, con el modelo teacher preentrenado en ImageNet. El modelo student usa la misma arquitectura, pero su red se inicializa aleatoriamente.

Como se muestra en la figura anterior, la funci√≥n de p√©rdida se calcula sobre las salidas de los 3 primeros bloques de capas de ResNet18. Un bloque de capas corresponde al conjunto de capas que operan a la misma resoluci√≥n de imagen. As√≠, el modelo student se entrena para replicar los feature maps del modelo teacher solo en estas 3 salidas. El puntaje de anomal√≠a tambi√©n se calcula sobre estas salidas.

La funci√≥n de p√©rdida utilizada es simplemente la p√©rdida MSE (error cuadr√°tico medio), que ya hemos visto. Esta p√©rdida se calcula para cada feature map y luego se suman para obtener la p√©rdida total.

Conjunto de datos#

El conjunto de datos utilizado en el art√≠culo es MVTEC AD. Contiene 15 categor√≠as, de las cuales 10 son objetos y 5 son texturas. Cada categor√≠a incluye aproximadamente 350 im√°genes sin defectos para el entrenamiento y alrededor de 100 im√°genes defectuosas para las pruebas.

Creaci√≥n de los modelos teacher y student#

Para nuestros modelos, usaremos la misma clase y especificaremos las particularidades de cada modelo mediante par√°metros. Para facilitar el uso de un backbone existente, utilizamos la biblioteca timm (PyTorch Image Models). Esta biblioteca es muy √∫til para acceder a backbones y modelos preentrenados, y tambi√©n ofrece flexibilidad para manipular la red.

class resnet18timm(nn.Module):
    def __init__(self,backbone_name="resnet18",out_indices=[1,2,3],pretrained=True):
        super(resnet18timm, self).__init__()     
        # Features only permet permet de ne r√©cup√©rer que les features et pas la sortie du r√©seau, out_indices permet de choisir les couches √† r√©cup√©rer
        self.feature_extractor = timm.create_model(backbone_name,pretrained=pretrained,features_only=True,out_indices=out_indices)
        if pretrained:
            # Si le mod√®le est pr√©-entrain√© (donc c'est le teacher), on g√®le les poids
            self.feature_extractor.eval() 
            for param in self.feature_extractor.parameters():
                param.requires_grad = False   
        
    def forward(self, x):
        features = self.feature_extractor(x)
        return features

Ahora podemos crear nuestro modelo teacher y student:

student=resnet18timm(backbone_name="resnet18",out_indices=[1,2,3],pretrained=False).to(device)
teacher=resnet18timm(backbone_name="resnet18",out_indices=[1,2,3],pretrained=True).to(device)

Funci√≥n de p√©rdida#

La funci√≥n de p√©rdida utiliza la distancia euclidiana (MSE), definida de la siguiente manera: \(D(I_1, I_2) = \sqrt{\sum_{i=1}^{m} \sum_{j=1}^{n} \left( I_1(i,j) - I_2(i,j) \right)^2}\), donde \(I_1\) e \(I_2\) son nuestras dos im√°genes.

Nuestra implementaci√≥n de la p√©rdida usa esta distancia para comparar los feature maps entre s√≠ para las 3 parejas de feature maps:

class loss_kdad:
  def __init__(self):
    pass
  # fs_list : liste des features du student et ft_list : liste des features du teacher
  def __call__(self,fs_list, ft_list):
    t_loss = 0
    N = len(fs_list)
    for i in range(N):
      fs = fs_list[i]
      ft = ft_list[i]
      _, _, h, w = fs.shape
      # Normaliser les features am√©liore les r√©sultats
      fs_norm = F.normalize(fs, p=2)
      ft_norm = F.normalize(ft, p=2)

      # Calcul de la distance euclidienne
      f_loss = 0.5 * (ft_norm - fs_norm) ** 2
      # On prend la moyenne de la loss sur tous les pixels
      f_loss = f_loss.sum() / (h * w)
      t_loss += f_loss

    return t_loss / N

for epoch in range(epochs):

        outputs_student = student(image)

        loss.backward()
        optimizer.step()

        outputs_teacher = teacher(image)
        loss = criterion(outputs_student,outputs_teacher)
        val_loss += loss.item()
    print("Epoch {} - train loss : {} - val loss : {}".format(epoch,train_loss/len(train_loader),val_loss/len(val_loader)))

Ahora podemos evaluar nuestro modelo. Para ello, utilizamos la m√©trica AUROC (consulta el m√≥dulo 9 para un repaso).

scores = []
test_imgs = []
gt_list = []
for sample in test_loader:
    label=sample['has_anomaly']
    image = sample['imageBase'].to(device)
    test_imgs.extend(image.cpu())
    gt_list.extend(label.cpu().numpy())
    with torch.set_grad_enabled(False):
        
        outputs_student = student(image)
        outputs_teacher = teacher(image) 

        # La fonction cal_anomaly_maps permet de calculer la carte d'anomalie (√ßa sera utile pour la visualisation apr√®s)
        score =cal_anomaly_maps(outputs_student,outputs_teacher,256) 
        
    scores.append(score)
scores = np.asarray(scores)
gt_list = np.asarray(gt_list)

from sklearn.metrics import roc_auc_score

map_scores = np.asarray(scores)

max_anomaly_score = map_scores.max()
min_anomaly_score = map_scores.min()
map_scores = (map_scores - min_anomaly_score) / (max_anomaly_score - min_anomaly_score)
img_scores = map_scores.reshape(map_scores.shape[0], -1).max(axis=1)
gt_list = np.asarray(gt_list)
img_roc_auc = roc_auc_score(gt_list, img_scores)
print(" image hazelnut ROCAUC : %.3f" % (img_roc_auc))
