Regresi√≥n Log√≠stica#

El Neurona Artificial#

Sumerg√°monos en el coraz√≥n del asunto: el neurona artificial.

La figura de abajo muestra c√≥mo funciona un neurona artificial:

Un neurona artificial recibe un vector \(\mathbf{x}=(x_1,x_2,...,x_n)\) como entrada. Cada elemento \(x_i\) del vector \(\mathbf{x}\) se multiplica por un peso \(w_i\), luego se calcula la suma y se a√±ade un sesgo \(b\). Esta suma se pasa entonces a trav√©s de una funci√≥n llamada funci√≥n de activaci√≥n \(\phi\). \(Output = \phi(\sum_{i=0}^{n} w_i x_i + b)\) Este proceso se llama neurona artificial porque se inspira en el funcionamiento de un neurona biol√≥gico.

Funciones de Activaci√≥n#

Funci√≥n de Heaviside: Originalmente, la primera neurona artificial (el perceptr√≥n) utilizaba una funci√≥n de umbral como funci√≥n de activaci√≥n. Esto permite tomar una decisi√≥n (0 o 1) basada en la suma ponderada y un umbral definido. \(heaviside(x) = \left\{ \begin{array}{ll} 1 & \text{si } x > 0 \\ 0 & \text{de lo contrario} \end{array} \right. \text{con } x=\sum_{i=0}^{n} w_i x_i + b\) Esta funci√≥n de activaci√≥n es efectiva para la clasificaci√≥n binaria, pero no funciona para m√∫ltiples clases. Adem√°s, la funci√≥n no es diferenciable, lo que complica el uso del algoritmo de descenso de gradiente para optimizar los pesos \(w_i\) de la neurona.

Las funciones de activaci√≥n recientes son mucho m√°s interesantes para entrenar redes neuronales usando descenso de gradiente. Primero, son diferenciables, lo que permite la aplicaci√≥n de nuestro algoritmo de descenso de gradiente. Segundo, son no lineales, lo que permite a las redes aprender representaciones complejas. Tambi√©n hay otras ventajas espec√≠ficas para cada funci√≥n de activaci√≥n.

Una de las funciones de activaci√≥n ‚Äúrecientes‚Äù es la funci√≥n sigmoide, que detallamos aqu√≠:

Funci√≥n Sigmoide: Otra funci√≥n de activaci√≥n particularmente interesante debido a su analog√≠a con la probabilidad es la funci√≥n sigmoide. Esta vez, la funci√≥n permite obtener un valor entre 0 y 1 usando la f√≥rmula: \(sigmoid(x) = \frac{1}{1 + e^{-x}} \text{ con } x=\sum_{i=0}^{n} w_i x_i + b\)

# Trac√© de la fonction sigmo√Øde
import numpy as np
import matplotlib.pyplot as plt

def sigmoid(x):
    return 1 / (1 + np.exp(-x))
x = np.linspace(-10, 10, 100)
y = sigmoid(x)

plt.plot(x, y)
plt.title('Fonction sigmo√Øde')
plt.xlabel('x')
plt.ylabel('sigmoid(x)')
plt.grid(True)
plt.show()

Esta funci√≥n es diferenciable, y su derivada es: \(sigmoid'(x) = sigmoid(x) \cdot (1 - f(x))\) Por lo tanto, podemos aplicar el descenso de gradiente a nuestra neurona artificial cuando usamos esta funci√≥n de activaci√≥n.

Hay muchas otras funciones de activaci√≥n cada una con sus propios usos. Las veremos en los siguientes cursos. (\(Tanh\), \(ReLU\), \(Softmax\))

Funci√≥n de Costo#

En nuestro ejemplo de descenso de gradiente, nuestro objetivo era encontrar el m√≠nimo de una funci√≥n. Estos son los escenarios donde el descenso de gradiente sobresale. Para nuestro nuevo problema, necesitamos encontrar una funci√≥n que, al minimizarse, mejore las predicciones.

En nuestra clasificaci√≥n binaria, \(y_{true}\) es 1 si el estudiante es admitido y 0 en caso contrario.

Nuestro objetivo es predecir si el estudiante es admitido o no prediciendo la salida \(pred\).

Durante el entrenamiento, queremos entrenar nuestro modelo de regresi√≥n log√≠stica para predecir \(pred \approx y_{true}\).

Para hacerlo, usamos la funci√≥n de verosimilitud negativa, que se expresa de la siguiente manera: \(\text{loss} = - \left( y_{\text{true}} \cdot \log(\text{pred}) + (1 - y_{\text{true}}) \cdot \log(1 - \text{pred}) \right)\)

Para m√°s detalles sobre la regresi√≥n log√≠stica y la p√©rdida de verosimilitud negativa, puedes consultar este enlace.

Lo importante es entender c√≥mo var√≠a esta funci√≥n en funci√≥n de nuestra predicci√≥n \(pred\) y la etiqueta \(y_{true}\).

Para esto, consideremos el caso donde la etiqueta es \(y_{true}=1\). Analicemos dos escenarios:

Si \(pred=0.9\), lo que significa que nuestro modelo predice un 90% de probabilidad de admisi√≥n (una buena predicci√≥n), entonces: \(\text{loss} = - \left( 1.0 \cdot \log(0.9) + (1 - 1.0) \cdot \log(1 - 0.9) \right)\) \(\text{loss} = - \left( 1.0 \cdot \log(0.9) + 0 \cdot \log(1 - 0.9) \right)\) \(\text{loss} = - \left( 1.0 \cdot \log(0.9)\right)\) \(\text{loss} = - \left( 1.0 \cdot \log(0.9)\right)\) \(\text{loss} = 0.046\) La p√©rdida es baja, lo cual es bueno porque la predicci√≥n es precisa.

Si \(pred=0.2\), lo que significa que nuestro modelo predice un 20% de probabilidad de admisi√≥n (una mala predicci√≥n), entonces: \(\text{loss} = - \left( 1.0 \cdot \log(0.2) + (1 - 1.0) \cdot \log(1 - 0.2) \right)\) \(\text{loss} = - \left( 1.0 \cdot \log(0.2) + 0 \cdot \log(1 - 0.2) \right)\) \(\text{loss} = - \left( 1.0 \cdot \log(0.2)\right)\) \(\text{loss} = - \left( 1.0 \cdot \log(0.2)\right)\) \(\text{loss} = 0.70\) La p√©rdida es alta, lo cual es bueno porque la predicci√≥n es pobre.

Para un caso donde \(y_{true}=0\), encontramos una p√©rdida baja cuando \(pred\) est√° cerca de 0 y una p√©rdida alta cuando \(pred\) est√° cerca de 1 (puedes hacer el c√°lculo para practicar si es necesario).

Calculando Derivadas#

Ahora que tenemos una funci√≥n para minimizar, es necesario calcular la derivada de esta funci√≥n con respecto a cada uno de los pesos \(w_0\), \(w_1\), \(w_2\) y \(b\).

Por lo tanto, debemos calcular \(\frac{\partial loss}{\partial w_0}\), \(\frac{\partial loss}{\partial w_1}\), \(\frac{\partial loss}{\partial w_2}\) y \(\frac{\partial loss}{\partial b}\).

Para los pesos \(w_0\), \(w_1\) y \(w_2\), la derivada se calcula de la misma manera.

Usando la regla de la cadena, para \(w_0\), tenemos: \(\frac{\partial loss}{\partial w_0} = \frac{\partial loss}{\partial pred} \cdot \frac{\partial pred}{\partial w_0}\)

Como recordatorio, nuestra predicci√≥n \(pred\) corresponde a la salida de nuestra regresi√≥n log√≠stica con la funci√≥n de activaci√≥n \(sigmoid\).

Para el primer t√©rmino, la derivada de la p√©rdida con respecto a \(pred\) da: \(\frac{\partial loss}{\partial pred} = -(\frac{y_{true}}{pred} - \frac{1-y_{true}}{1-pred}) \) El c√°lculo no se detallar√° aqu√≠, pero puedes hacerlo t√∫ mismo para verificar.

Para el segundo t√©rmino, la derivada de \(pred\) con respecto a \(w_0\) da: \(\frac{\partial pred}{\partial w_0} = pred \cdot (1-pred) \cdot x_0\)

Combinando los dos t√©rminos, obtenemos: \(\frac{\partial loss}{\partial w_0} =-(\frac{y_{true}}{pred} - \frac{1-y_{true}}{1-pred}) \cdot pred \cdot (1-pred) \cdot x_0\)

Y despu√©s de simplificaci√≥n (m√°gicamente), \(\frac{\partial loss}{\partial w_0} = (pred-y_{true}) \cdot x_0\)

Sin detallar el c√°lculo, tambi√©n obtenemos: \(\frac{\partial loss}{\partial b} = pred-y_{true}\)

Regresi√≥n Log√≠stica#

Ahora que tenemos todos los elementos, definamos nuestra funci√≥n de regresi√≥n log√≠stica en Python:

# Notre classe de regression logistique
class logistic_regression():
  def __init__(self) -> None:
    self.w0=np.random.randn()
    self.w1=np.random.randn()
    self.w2=np.random.randn()
    self.b=0
  def __call__(self,x0,x1,x2):
    # Somme pond√©r√©e et ajout du biais
    pond=self.w0*x0+self.w1*x1+self.w2*x2+self.b
    # Application de la sigmo√Øde
    pred=sigmoid(pond)
    return pred
    
def loss(y_true, y_pred):
  # Calcul du loss (log vraisemblance n√©gative)
  loss = - (y_true * np.log(y_pred) + (1 - y_true) * np.log(1 - y_pred))
  return loss
    

def update_weights(model,pred, x0, x1, x2, y_true, learning_rate):
       
  # On calcule les d√©riv√©es en fonction des poids et du biais 
  dL_dw0 = (pred - y_true) * x0
  dL_dw1 = (pred - y_true) * x1
  dL_dw2 = (pred - y_true) * x2
  dL_db = pred - y_true
      
  # On modifie les param√®tres pour r√©duire le loss 
  # La modification des poids d√©pend du learning rate, du signe de la d√©riv√©e et de la valeur de la d√©riv√©e
  model.w0 -= learning_rate * dL_dw0
  model.w1 -= learning_rate * dL_dw1
  model.w2 -= learning_rate * dL_dw2
  model.b -= learning_rate * dL_db

# Initialisation du mod√®le et des hyperparam√®tres
learning_rate = 0.01
epochs = 1000 # le nombre d'it√©rations d'entrainement
model = logistic_regression()
