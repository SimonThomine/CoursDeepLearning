El Neurona Artificial#

SumergÃ¡monos en el corazÃ³n del asunto: el neurona artificial.

La figura de abajo muestra cÃ³mo funciona un neurona artificial:

Un neurona artificial recibe un vector \(\mathbf{x}=(x_1,x_2,...,x_n)\) como entrada. Cada elemento \(x_i\) del vector \(\mathbf{x}\) se multiplica por un peso \(w_i\), luego se calcula la suma y se aÃ±ade un sesgo \(b\). Esta suma se pasa entonces a travÃ©s de una funciÃ³n llamada funciÃ³n de activaciÃ³n \(\phi\). \(Output = \phi(\sum_{i=0}^{n} w_i x_i + b)\) Este proceso se llama neurona artificial porque se inspira en el funcionamiento de un neurona biolÃ³gico.

Funciones de ActivaciÃ³n#

FunciÃ³n de Heaviside: Originalmente, la primera neurona artificial (el perceptrÃ³n) utilizaba una funciÃ³n de umbral como funciÃ³n de activaciÃ³n. Esto permite tomar una decisiÃ³n (0 o 1) basada en la suma ponderada y un umbral definido. \(heaviside(x) = \left\{ \begin{array}{ll} 1 & \text{si } x > 0 \\ 0 & \text{de lo contrario} \end{array} \right. \text{con } x=\sum_{i=0}^{n} w_i x_i + b\) Esta funciÃ³n de activaciÃ³n es efectiva para la clasificaciÃ³n binaria, pero no funciona para mÃºltiples clases. AdemÃ¡s, la funciÃ³n no es diferenciable, lo que complica el uso del algoritmo de descenso de gradiente para optimizar los pesos \(w_i\) de la neurona.

Las funciones de activaciÃ³n recientes son mucho mÃ¡s interesantes para entrenar redes neuronales usando descenso de gradiente. Primero, son diferenciables, lo que permite la aplicaciÃ³n de nuestro algoritmo de descenso de gradiente. Segundo, son no lineales, lo que permite a las redes aprender representaciones complejas. TambiÃ©n hay otras ventajas especÃ­ficas para cada funciÃ³n de activaciÃ³n.

Una de las funciones de activaciÃ³n â€œrecientesâ€ es la funciÃ³n sigmoide, que detallamos aquÃ­:

FunciÃ³n Sigmoide: Otra funciÃ³n de activaciÃ³n particularmente interesante debido a su analogÃ­a con la probabilidad es la funciÃ³n sigmoide. Esta vez, la funciÃ³n permite obtener un valor entre 0 y 1 usando la fÃ³rmula: \(sigmoid(x) = \frac{1}{1 + e^{-x}} \text{ con } x=\sum_{i=0}^{n} w_i x_i + b\)

FunciÃ³n de Costo#

En nuestro ejemplo de descenso de gradiente, nuestro objetivo era encontrar el mÃ­nimo de una funciÃ³n. Estos son los escenarios donde el descenso de gradiente sobresale. Para nuestro nuevo problema, necesitamos encontrar una funciÃ³n que, al minimizarse, mejore las predicciones.

En nuestra clasificaciÃ³n binaria, \(y_{true}\) es 1 si el estudiante es admitido y 0 en caso contrario.

Nuestro objetivo es predecir si el estudiante es admitido o no prediciendo la salida \(pred\).

Durante el entrenamiento, queremos entrenar nuestro modelo de regresiÃ³n logÃ­stica para predecir \(pred \approx y_{true}\).

Para hacerlo, usamos la funciÃ³n de verosimilitud negativa, que se expresa de la siguiente manera: \(\text{loss} = - \left( y_{\text{true}} \cdot \log(\text{pred}) + (1 - y_{\text{true}}) \cdot \log(1 - \text{pred}) \right)\)

Para mÃ¡s detalles sobre la regresiÃ³n logÃ­stica y la pÃ©rdida de verosimilitud negativa, puedes consultar este enlace.

Lo importante es entender cÃ³mo varÃ­a esta funciÃ³n en funciÃ³n de nuestra predicciÃ³n \(pred\) y la etiqueta \(y_{true}\).

Para esto, consideremos el caso donde la etiqueta es \(y_{true}=1\). Analicemos dos escenarios:

Si \(pred=0.9\), lo que significa que nuestro modelo predice un 90% de probabilidad de admisiÃ³n (una buena predicciÃ³n), entonces: \(\text{loss} = - \left( 1.0 \cdot \log(0.9) + (1 - 1.0) \cdot \log(1 - 0.9) \right)\) \(\text{loss} = - \left( 1.0 \cdot \log(0.9) + 0 \cdot \log(1 - 0.9) \right)\) \(\text{loss} = - \left( 1.0 \cdot \log(0.9)\right)\) \(\text{loss} = - \left( 1.0 \cdot \log(0.9)\right)\) \(\text{loss} = 0.046\) La pÃ©rdida es baja, lo cual es bueno porque la predicciÃ³n es precisa.

Si \(pred=0.2\), lo que significa que nuestro modelo predice un 20% de probabilidad de admisiÃ³n (una mala predicciÃ³n), entonces: \(\text{loss} = - \left( 1.0 \cdot \log(0.2) + (1 - 1.0) \cdot \log(1 - 0.2) \right)\) \(\text{loss} = - \left( 1.0 \cdot \log(0.2) + 0 \cdot \log(1 - 0.2) \right)\) \(\text{loss} = - \left( 1.0 \cdot \log(0.2)\right)\) \(\text{loss} = - \left( 1.0 \cdot \log(0.2)\right)\) \(\text{loss} = 0.70\) La pÃ©rdida es alta, lo cual es bueno porque la predicciÃ³n es pobre.

Para un caso donde \(y_{true}=0\), encontramos una pÃ©rdida baja cuando \(pred\) estÃ¡ cerca de 0 y una pÃ©rdida alta cuando \(pred\) estÃ¡ cerca de 1 (puedes hacer el cÃ¡lculo para practicar si es necesario).

Calculando Derivadas#

Ahora que tenemos una funciÃ³n para minimizar, es necesario calcular la derivada de esta funciÃ³n con respecto a cada uno de los pesos \(w_0\), \(w_1\), \(w_2\) y \(b\).

Por lo tanto, debemos calcular \(\frac{\partial loss}{\partial w_0}\), \(\frac{\partial loss}{\partial w_1}\), \(\frac{\partial loss}{\partial w_2}\) y \(\frac{\partial loss}{\partial b}\).

Para los pesos \(w_0\), \(w_1\) y \(w_2\), la derivada se calcula de la misma manera.

Usando la regla de la cadena, para \(w_0\), tenemos: \(\frac{\partial loss}{\partial w_0} = \frac{\partial loss}{\partial pred} \cdot \frac{\partial pred}{\partial w_0}\)

Como recordatorio, nuestra predicciÃ³n \(pred\) corresponde a la salida de nuestra regresiÃ³n logÃ­stica con la funciÃ³n de activaciÃ³n \(sigmoid\).

Para el primer tÃ©rmino, la derivada de la pÃ©rdida con respecto a \(pred\) da: \(\frac{\partial loss}{\partial pred} = -(\frac{y_{true}}{pred} - \frac{1-y_{true}}{1-pred}) \) El cÃ¡lculo no se detallarÃ¡ aquÃ­, pero puedes hacerlo tÃº mismo para verificar.

Para el segundo tÃ©rmino, la derivada de \(pred\) con respecto a \(w_0\) da: \(\frac{\partial pred}{\partial w_0} = pred \cdot (1-pred) \cdot x_0\)

Combinando los dos tÃ©rminos, obtenemos: \(\frac{\partial loss}{\partial w_0} =-(\frac{y_{true}}{pred} - \frac{1-y_{true}}{1-pred}) \cdot pred \cdot (1-pred) \cdot x_0\)

Y despuÃ©s de simplificaciÃ³n (mÃ¡gicamente), \(\frac{\partial loss}{\partial w_0} = (pred-y_{true}) \cdot x_0\)

Sin detallar el cÃ¡lculo, tambiÃ©n obtenemos: \(\frac{\partial loss}{\partial b} = pred-y_{true}\)

# Notre classe de regression logistique
class logistic_regression():
  def __init__(self) -> None:
    self.w0=np.random.randn()
    self.w1=np.random.randn()
    self.w2=np.random.randn()
    self.b=0
  def __call__(self,x0,x1,x2):
    # Somme pondÃ©rÃ©e et ajout du biais
    pond=self.w0*x0+self.w1*x1+self.w2*x2+self.b
    # Application de la sigmoÃ¯de
    pred=sigmoid(pond)
    return pred
    
def loss(y_true, y_pred):
  # Calcul du loss (log vraisemblance nÃ©gative)
  loss = - (y_true * np.log(y_pred) + (1 - y_true) * np.log(1 - y_pred))
  return loss
    

def update_weights(model,pred, x0, x1, x2, y_true, learning_rate):
       
  # On calcule les dÃ©rivÃ©es en fonction des poids et du biais 
  dL_dw0 = (pred - y_true) * x0
  dL_dw1 = (pred - y_true) * x1
  dL_dw2 = (pred - y_true) * x2
  dL_db = pred - y_true
      
  # On modifie les paramÃ¨tres pour rÃ©duire le loss 
  # La modification des poids dÃ©pend du learning rate, du signe de la dÃ©rivÃ©e et de la valeur de la dÃ©rivÃ©e
  model.w0 -= learning_rate * dL_dw0
  model.w1 -= learning_rate * dL_dw1
  model.w2 -= learning_rate * dL_dw2
  model.b -= learning_rate * dL_db

# Initialisation du modÃ¨le et des hyperparamÃ¨tres
learning_rate = 0.01
epochs = 1000 # le nombre d'itÃ©rations d'entrainement
model = logistic_regression()
