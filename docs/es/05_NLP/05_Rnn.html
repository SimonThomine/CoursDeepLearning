Redes neuronales recurrentes#

En este curso, exploraremos las redes neuronales recurrentes (RNN) para predecir el siguiente car√°cter. Nos basamos en la arquitectura descrita en el art√≠culo Recurrent neural network based language model, que propone una versi√≥n simplificada de RNN para esta tarea.

Ventaja clave: a diferencia de los modelos basados en redes fully connected (vistos anteriormente), los RNN no requieren un tama√±o de contexto fijo.

Los RNN mantenienen en memoria el contexto, independientemente de la longitud de la secuencia. Aunque esto suena prometedor en teor√≠a, veremos al final del curso que tienen limitaciones pr√°cticas.

Figura extra√≠da del art√≠culo original.

¬øC√≥mo funciona una RNN?#

Las RNN procesan los datos de forma secuencial: los caracteres se analizan uno a uno. El car√°cter siguiente depende tanto del elemento actual como del estado oculto (\(s\)), que almacena informaci√≥n de los caracteres anteriores.

Componentes matem√°ticos (en un instante \(t\)):

  • Entrada (\(x\)): combinaci√≥n del one-hot encoding del car√°cter actual (\(w(t)\)) y el estado previo (\(s(t-1)\)). \(x(t) = w(t) + s(t-1)\)

  • Estado oculto (\(s\)): se actualiza aplicando la funci√≥n sigmoide a la entrada. \(s(t) = sigmoid(x(t))\)

  • Salida (\(y\)): probabilidades de los caracteres siguientes, calculadas con softmax. \(y(t) = softmax(s(t))\)

Par√°metro clave: el tama√±o de la capa oculta (\(s\)). Inicializaci√≥n: \(s(0)\) suele ser un vector peque√±o (ej. ceros o valores aleatorios).

Construcci√≥n del modelo#

¬°Construyamos el modelo!

Seg√∫n el art√≠culo, la entrada (car√°cter) se codifica en one-hot y se suma al estado previo. Necesitamos dos capas fully connected:

  1. Primera capa: transforma \(x(t)\) en el estado oculto \(s(t)\).

  2. Segunda capa: convierte \(s(t)\) en la predicci√≥n \(y(t)\).

Ecuaci√≥n del art√≠culo: \(f\) = sigmoide, \(g\) = softmax.

Recomendaci√≥n: El art√≠culo original es claro y conciso. ¬°Vale la pena leerlo!

class rnn(nn.Module): 
  def __init__(self,hidden_dim,vocab_size) -> None:
    super(rnn, self).__init__()
    self.hidden_to_hidden=nn.Linear(hidden_dim+vocab_size, hidden_dim)
    self.hidden_to_output=nn.Linear(hidden_dim, vocab_size)
    self.vocab_size=vocab_size
    self.hidden_dim=hidden_dim
    self.sigmoid=nn.Sigmoid() 
    
  # Le r√©seau prend en entr√©e le caract√®re actuel et le state pr√©c√©dent
  def forward(self, x,state):
    # On one-hot encode le caract√®re
    x = torch.nn.functional.one_hot(x, self.vocab_size).float()
    if state is None:
      # Si on a pas de state (d√©but de la s√©quence), on initialise le state avec des petites valeurs al√©atoires
      state = torch.randn(self.hidden_dim) * 0.1
    x = torch.cat((x, state), dim=-1)  # Concat√©nation de x et du state
    state = self.sigmoid(self.hidden_to_hidden(x)) # Calcul du nouveau state
    output = self.hidden_to_output(state) # Calcul de l'output
    # On renvoie l'output et le state pour le prochain pas de temps
    return output, state.detach() # detach() pour √©viter de propager le gradient dans le state

Limitaciones de las RNN#

Aunque las RNN fueron centrales en NLP y deep learning, tienen limitaciones que las hacen poco pr√°cticas para modelos grandes:

  • Contexto te√≥ricamente infinito, pero su estructura secuencial dificulta propagar informaci√≥n en secuencias largas.

  • Problema del vanishing gradient: los gradientes se desvanecen en secuencias largas.

  • Dificultad para paralelizar: los GPU est√°n optimizados para c√°lculos paralelos, pero las RNN son inherentemente secuenciales (entrenamiento m√°s lento).

  • Estructura fija: no siempre captura relaciones complejas entre datos.

Desde la llegada de los transformers, el uso de RNN ha disminuido significativamente.
