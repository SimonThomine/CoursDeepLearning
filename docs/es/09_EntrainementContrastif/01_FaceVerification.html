Verificaci√≥n facial#

La verificaci√≥n facial consiste en confirmar la identidad de una persona a partir de una foto de referencia. Por ejemplo, una empresa puede almacenar una foto de cada empleado en una base de datos. Cuando un empleado se presenta en la entrada con su tarjeta de identificaci√≥n, el sistema compara su foto con la registrada para verificar su identidad.

Esta tarea presenta varios desaf√≠os:

  • No siempre se tiene acceso a m√∫ltiples fotos de cada empleado para el entrenamiento.

  • Es preferible evitar reentrenar el modelo cada vez que un nuevo empleado se incorpora a la empresa.

Para resolver este problema, utilizaremos el m√©todo descrito en el art√≠culo Aprendizaje de una m√©trica de similitud discriminativa, con aplicaci√≥n a la verificaci√≥n facial. Este m√©todo entrena una medida de similitud a partir de los datos.

Es particularmente efectivo para conjuntos de datos que contienen un gran n√∫mero de categor√≠as (cada persona es una categor√≠a para los rostros) y/o cuando no todas las categor√≠as est√°n disponibles durante el entrenamiento (nuevos empleados).

Comencemos analizando algunos puntos te√≥ricos del art√≠culo para comprender el funcionamiento del m√©todo y luego pasaremos a la implementaci√≥n.

An√°lisis te√≥rico del art√≠culo#

Intuici√≥n#

La idea principal del art√≠culo es entrenar el modelo de manera contrastiva utilizando pares de im√°genes. Dos escenarios son posibles:

  • Las dos im√°genes provienen de la misma categor√≠a.

  • Las dos im√°genes provienen de categor√≠as diferentes.

El objetivo es tener un modelo \(G_w\) que mapea una imagen en un espacio latente donde la distancia euclidiana (o coseno) permite diferenciar los elementos de manera efectiva.

M√°s formalmente, queremos que: \(D_W(X_1,X_2) = \lVert G_W(X_1) - G_W(X_2)\rVert\) sea peque√±a si \(X_1\) y \(X_2\) est√°n en la misma categor√≠a y grande si est√°n en categor√≠as diferentes.

El modelo se entrena en pares de im√°genes, y los pesos de \(G_W\) son compartidos.

La arquitectura se parece a esto:

Funci√≥n de p√©rdida contrastiva#

¬øC√≥mo definir nuestra funci√≥n de p√©rdida para este objetivo? Tomemos \(Y\) como la etiqueta de nuestro par de im√°genes. Si \(Y\) vale 0, las im√°genes provienen de la misma categor√≠a; de lo contrario, \(Y\) vale 1.

La funci√≥n de p√©rdida contrastiva se expresa de la siguiente manera: \(L(Y,X_1,X_2)= (1-Y)\frac{1}{2}(D_W)^2 + (Y)\frac{1}{2}(max(0,m-D_W))¬≤\)

Esta funci√≥n puede parecer compleja, pero en realidad es bastante simple. Examinemos los dos valores posibles de \(Y\):

  • Si \(Y = 0\), entonces \(L(Y,X_1,X_2)= \frac{1}{2}(D_W)^2\), lo que corresponde simplemente a la distancia \(D_W\) definida previamente.

  • Si \(Y = 1\), entonces \(L(Y,X_1,X_2)= \frac{1}{2}(max(0,m-D_W))¬≤\). Esta parte requiere una explicaci√≥n adicional. Podr√≠amos suponer que simplemente usar \(\frac{1}{D_W}\) permitir√≠a maximizar la distancia (que es lo que queremos). En la pr√°ctica, esto no funciona porque el modelo podr√≠a aprender a separar los elementos con un margen muy peque√±o, lo que podr√≠a causar problemas de generalizaci√≥n. El par√°metro \(m\) (margen) permite especificar un margen m√≠nimo de separaci√≥n entre los elementos diferentes. Esto hace que el modelo sea m√°s robusto y estabiliza el entrenamiento.

Nota: ¬øPor qu√© no simplemente minimizar \(D_W(X_1,X_2)\) en pares de im√°genes de la misma categor√≠a √∫nicamente? Si hacemos esto, el modelo puede ‚Äúcolapsar‚Äù y aprender una soluci√≥n trivial que consiste en hacer de \(G_W\) una funci√≥n constante, de manera que la p√©rdida siempre ser√° igual a 0. En la pr√°ctica, el modelo siempre buscar√° la soluci√≥n m√°s simple, por lo que es importante definir bien sus funciones de p√©rdida para evitar que el modelo encuentre un atajo trivial.

Arquitectura del modelo y entrenamiento#

\(G_W\) puede ser cualquier red neuronal. El art√≠culo propone utilizar una red convolucional por sus capacidades con im√°genes y su robustez frente a distorsiones geom√©tricas (frecuentes en la verificaci√≥n facial).

No reproduciremos estrictamente la arquitectura del art√≠culo, sino que dise√±aremos nuestra propia arquitectura y la entrenaremos con un conjunto de datos m√°s extenso que los del art√≠culo.

Para el entrenamiento, el art√≠culo recomienda utilizar un 50% de pares positivos y un 50% de pares negativos.

Funci√≥n de p√©rdida contrastiva#

Es hora de definir nuestra funci√≥n de p√©rdida. Para ello, simplemente retomaremos la f√≥rmula que introdujimos anteriormente. Para \(D_W\), tomamos la distancia euclidiana: \(D_W (X_1,X_2) = \lVert X_1 - X_2 \rVert_2 = \sqrt{\sum_{i=1}^{n} (X_{1,i} - X_{2,i})^2}\)

class ContrastiveLoss(nn.Module):
    def __init__(self, margin=1.0):
        super(ContrastiveLoss, self).__init__()
        self.margin = margin

    def forward(self, output1, output2, label):
        # Distance euclidienne entre les deux sorties du r√©seau
        euclidean_distance = F.pairwise_distance(output1, output2, keepdim=True)
        
        # Calcul du loss contrastif
        loss_contrastive = torch.mean(
            (1 - label) * torch.pow(euclidean_distance, 2) +
            (label) * torch.pow(torch.clamp(self.margin - euclidean_distance, min=0.0), 2)
        )
        
        return loss_contrastive

Creaci√≥n de nuestro modelo#

Para esta tarea, construiremos una red convolucional cl√°sica y relativamente peque√±a. El objetivo del modelo es proyectar los datos en un espacio donde la distancia euclidiana sea relevante.

class SiameseNetwork(torch.nn.Module):

    def __init__(self):
        super(SiameseNetwork, self).__init__()

        # Extraction des features pertinentes
        self.conv_net = nn.Sequential(
            nn.Conv2d(3, 16, kernel_size=7, stride=2, padding=3),
            nn.ReLU(),
            nn.Conv2d(16, 32, kernel_size=5, stride=2, padding=2),
            nn.ReLU(),
            nn.Conv2d(32, 64, kernel_size=5, stride=2, padding=2),
            nn.ReLU(),
            nn.Conv2d(64, 128, kernel_size=3, stride=2, padding=1),
            nn.ReLU(),
            nn.MaxPool2d(2, 2),
        )
        
        # Projection des features dans un espace de dimension r√©duite
        self.linear=nn.Sequential(
            nn.Linear(128 * 4 * 4, 256),
            nn.ReLU(),
            nn.Linear(256, 32), 
            )
    
    def forward(self, x1, x2):
        x1 = self.conv_net(x1)
        x2 = self.conv_net(x2)
        x1 = x1.view(x1.size(0), -1)
        x2 = x2.view(x2.size(0), -1)
        x1=self.linear(x1)
        x2=self.linear(x2)
        return x1,x2

Definamos nuestros hiperpar√°metros de entrenamiento:

epochs = 30

margin = 1.0
model = SiameseNetwork().to(device)
optimizer = torch.optim.SGD(model.parameters(), lr=lr)

for epoch in range(epochs):
  loss_train = 0
  for x1, x2, label in train_loader:
    x1, x2, label = x1.to(device), x2.to(device), label.to(device)

    loss_train+=loss.item()
    loss.backward()
    optimizer.step()
  print(f"Epoch {epoch+1}/{epochs} Loss: {loss_train/len(train_loader)}")

Evaluaci√≥n del modelo#

Ahora que el entrenamiento ha finalizado, es necesario evaluar el modelo. Pero, ¬øc√≥mo hacerlo? Este no es el tipo de modelo al que estamos acostumbrados a evaluar.

Tenemos nuestro conjunto de datos de prueba y podemos calcular las distancias entre cada par de este conjunto. De esta manera, podemos obtener una puntuaci√≥n de similitud para cada elemento del conjunto de datos de prueba.

La curva ROC: La curva ROC (Receiver Operating Characteristic) es un gr√°fico que ilustra el rendimiento de un modelo de clasificaci√≥n binaria (positivo o negativo en nuestro caso) a diferentes umbrales de decisi√≥n. La curva ROC contiene los siguientes elementos:

  • Eje X: la tasa de falsos positivos (o tasa de falsas alarmas). Es el n√∫mero de elementos negativos incorrectamente clasificados como positivos, dividido por el n√∫mero total de elementos negativos.

  • Eje Y: la tasa de verdaderos positivos (recall o sensibilidad). Es el n√∫mero de elementos positivos correctamente clasificados como positivos, dividido por el n√∫mero total de elementos positivos.

Cada punto de la curva representa un umbral de decisi√≥n diferente para la clasificaci√≥n de los elementos.

Figura extra√≠da de blogpost.

Para juzgar la calidad de un modelo, podemos calcular el √°rea bajo la curva ROC (AUROC). En el caso de un clasificador aleatorio, se obtiene un AUROC de 0.5, mientras que para un clasificador perfecto, el AUROC es de 1.

Podemos calcularlo para nuestro modelo:

from sklearn.metrics import roc_curve, auc

model.eval()
labels = []
distances = []
with torch.no_grad():
    for x1, x2, label in test_loader:
        x1, x2, label = x1.to(device), x2.to(device), label.to(device)
        out1, out2 = model(x1, x2)
        dist = torch.nn.functional.pairwise_distance(out1, out2)
        distances.extend(dist.cpu().numpy())
        labels.extend(label.cpu().numpy())
        
fpr, tpr, thresholds = roc_curve(labels, distances)
roc_auc = auc(fpr, tpr)
print(f"ROC AUC: {roc_auc}")

P√©rdida por tripletes#

Reconocimiento facial#

En el campo del procesamiento de rostros, existen dos categor√≠as principales: la verificaci√≥n facial y el reconocimiento facial.

Para entender la diferencia entre ambas, retomemos el caso de nuestro empleado en una empresa:

  • Verificaci√≥n facial: En el caso de la verificaci√≥n facial, un empleado quiere entrar al edificio con su tarjeta de identificaci√≥n. Pasa su tarjeta por la m√°quina, y el modelo de verificaci√≥n facial verifica que se trata efectivamente del empleado que posee esa tarjeta.

  • Reconocimiento facial: Aqu√≠, debemos imaginar que el empleado se presenta al edificio sin su tarjeta y que el modelo es capaz de reconocerlo entre la base de datos de todos los empleados para permitirle la entrada al edificio.

Como pueden imaginar, la segunda tarea es m√°s compleja, tanto en t√©rminos de dificultad pura como en t√©rminos de restricciones de tiempo de procesamiento (no podemos hacer esperar al empleado en la puerta durante una hora mientras comparamos su foto con las cientos de fotos de nuestra base de datos).

FaceNet#

Para la tarea de reconocimiento facial, un art√≠culo FaceNet: A Unified Embedding for Face Recognition and Clustering introdujo un nuevo tipo de p√©rdida que se asemeja a la p√©rdida contrastiva. Se trata de la p√©rdida por tripletes, que describiremos brevemente.

El objetivo de esta p√©rdida es similar al de la p√©rdida contrastiva: busca aprender una representaci√≥n de manera que los vectores de caracter√≠sticas (o embeddings) de puntos similares est√©n cercanos en el espacio latente, mientras que los de puntos diferentes est√©n alejados entre s√≠.

A diferencia de la p√©rdida contrastiva, que se basa en pares de ejemplos, la p√©rdida por tripletes se basa en tripletes de ejemplos.

Cada ejemplo tiene un rol particular:

  • Anchor (A): El ejemplo de referencia del cual queremos aprender una representaci√≥n.

  • Positive (P): Un ejemplo similar al anchor (por ejemplo, la misma persona).

  • Negative (N): Un ejemplo diferente al anchor (una persona diferente).

La funci√≥n de p√©rdida se define de la siguiente manera: \(L_{\text{triplet}} = \max\left( d(A, P) - d(A, N) + \alpha, 0 \right)\) donde \(d\) es la distancia y \(\alpha\) el margen (margin).

Esta p√©rdida minimiza la distancia entre \(P\) y \(A\) mientras maximiza la distancia entre \(N\) y \(A\).

Figura extra√≠da del art√≠culo original.

Esto puede parecer redundante con la p√©rdida contrastiva, y en parte lo es. Sin embargo, la p√©rdida por tripletes ha demostrado mejores resultados cuando se trata de hacer una distinci√≥n fina entre clases bastante similares (t√≠picamente el caso del reconocimiento facial). Hoy en d√≠a, su uso es preferido sobre la p√©rdida contrastiva para tareas de procesamiento de rostros.
