On peut maintenant impl√©menter nos deux mod√®les. Commen√ßons par regarder les sp√©cificit√©s d‚Äôarchitecture d√©crites dans le papier.

Note : Dans le papier, les auteurs disent utiliser des fractional-strided convolutions. Il s‚Äôagit en fait de convolutions transpos√©es et le terme fractional-strided convolutions n‚Äôest plus vraiment utilis√© de nos jours.

def convT_bn_relu(in_channels, out_channels, kernel_size, stride, padding):
  return nn.Sequential(
    nn.ConvTranspose2d(in_channels, out_channels, kernel_size, stride, padding,bias=False),
    nn.BatchNorm2d(out_channels),
    nn.ReLU()
  )

class generator(nn.Module):
  def __init__(self, z_dim=100,features_g=64):
    super(generator, self).__init__()
    self.gen = nn.Sequential(
      convT_bn_relu(z_dim, features_g*8, kernel_size=4, stride=1, padding=0),
      convT_bn_relu(features_g*8, features_g*4, kernel_size=4, stride=2, padding=1),
      convT_bn_relu(features_g*4, features_g*2, kernel_size=4, stride=2, padding=1),
      nn.ConvTranspose2d(features_g*2, 1, kernel_size=4, stride=2, padding=1),
      nn.Tanh()
    )
  def forward(self, x):
    return self.gen(x)

Le papier ne d√©crit pas directement l‚Äôarchitecture du discriminateur. On va globalement reprendre l‚Äôarchitecture du g√©n√©rateur mais dans l‚Äôautre sens.

def conv_bn_lrelu(in_channels, out_channels, kernel_size, stride, padding):
  return nn.Sequential(
    nn.Conv2d(in_channels, out_channels, kernel_size, stride, padding,bias=False),
    nn.BatchNorm2d(out_channels),
    nn.LeakyReLU()
  )

class discriminator(nn.Module):
  def __init__(self, features_d=64) -> None:
    super().__init__()
    self.discr = nn.Sequential(
      conv_bn_lrelu(1, features_d, kernel_size=3, stride=2, padding=1),
      conv_bn_lrelu(features_d, features_d*2, kernel_size=3, stride=2, padding=1),
      conv_bn_lrelu(features_d*2, features_d*4, kernel_size=3, stride=2, padding=1),
      nn.Conv2d(256, 1, kernel_size=3, stride=2, padding=0),
      nn.Sigmoid()
    )
    
  def forward(self, x):
    return self.discr(x)

Il est temps de passer aux choses s√©rieuses. La boucle d‚Äôentra√Ænement d‚Äôun GAN est bien plus complexe que les boucles d‚Äôentra√Ænement des mod√®les qu‚Äôon a vus jusqu‚Äô√† pr√©sent. Commen√ßons par d√©finir nos hyperparam√®tres d‚Äôentra√Ænement et par initialiser nos mod√®les :

On va √©galement cr√©er un bruit fixed_noise pour √©valuer visuellement notre mod√®le √† chaque √©tape d‚Äôentra√Ænement.

Avant de construire notre boucle d‚Äôentra√Ænement, r√©capitulons les √©tapes qu‚Äôon doit faire :

  • On commence par r√©cup√©rer batch_size √©l√©ments du dataset d‚Äôentra√Ænement et on pr√©dit les labels avec notre discriminateur

  • Ensuite, on g√©n√®re batch_size √©l√©ments avec notre g√©n√©rateur et on pr√©dit les labels

  • On va ensuite mettre √† jour les poids du mod√®le discriminateur √† partir des deux loss

  • On va ensuite √† nouveau pr√©dire les labels des donn√©es g√©n√©r√©es (car on a mis √† jour le discriminateur entre temps)

  • √Ä partir de ces valeurs, on calcule le loss et on met √† jour notre g√©n√©rateur

for epoch in range(epochs):
  lossD_epoch = 0
  lossG_epoch = 0
  for real_images,_ in train_loader:
    real_images=real_images.to(device)
    pred_real = disc(real_images).view(-1)
    lossD_real = criterion(pred_real, torch.ones_like(pred_real)) # Les labels sont 1 pour les vraies images
    
    batch_size = real_images.shape[0]

    lossD=lossD_real + lossD_fake

    disc.zero_grad()
    lossD.backward()
    opt_disc.step()
    
    # On refait l'inf√©rence pour les images g√©n√©r√©es (avec le discriminateur mis √† jour)

    lossG_epoch += lossG.item()
    gen.zero_grad()
    lossG.backward()
    opt_gen.step()
    
  # On g√©n√®re des images avec le g√©n√©rateur
  if epoch % 10 == 0 or epoch==0:

    gen.eval()
