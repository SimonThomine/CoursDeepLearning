Descente de gradient stochastique (SGD)#

Avant de passer √† la suite de notre code, il est n√©cessaire de faire un point sur la descente de gradient stochastique.
Pour appliquer l‚Äôalgorithme de descente de gradient sur un dataset de taille \(N\), il faudrait en th√©orie calculer la perte et le gradient sur chacun des √©l√©ments avant de mettre √† jour les poids. Cette m√©thode a l‚Äôavantage de garantir une diminution de la perte √† chaque it√©ration de l‚Äôentra√Ænement mais est tr√®s couteuse pour des datasets o√π \(N\) est grand. En pratique, on a souvent \(N>10‚Å∂\). De plus, il faudrait garder les valeurs des gradients de l‚Äôensemble des \(N\) √©l√©ments en m√©moire ce qui est impossible pour des ensembles de donn√©es trop important.

Pour palier √† ce probl√®me, nous utilisons des mini-batch qui correspondent √† des groupes d‚Äô√©chantillons du dataset. L‚Äôoptimisation se fait de la m√™me mani√®re que sur la descente de gradient sauf que la mise √† jour des poids se fait √† chaque mini-batch (donc beaucoup plus fr√©quemment). Le processus d‚Äôoptimisation est alors beaucoup plus rapide et permet de traiter des grandes quantit√©s de donn√©es. A titre indicatif, le param√®tre d√©finissant la taille d‚Äôun mini-batch est appel√© batch size et sa valeur est souvent 16, 32 ou 64.
Pour en apprendre plus sur la descente du gradient stochastique : wikipedia ou blogpost

D√©finissons, en python, une fonction pour r√©cuperer batch_size √©l√©ments al√©atoires de notre dataset.

def get_batch(batch_size=64):
  ri = np.random.permutation(X.shape[0])[:batch_size]
  Xb, labels = X[ri], y[ri]
  #inputs = [list(map(Value, xrow)) for xrow in Xb] # OLD
  # Conversion des inputs en Value pour pouvoir utiliser micrograd
  inputs = [list([Value(xrow[0]),Value(xrow[1])]) for xrow in Xb]
  return inputs,labels

Fonction de perte#

Pour entra√Æner notre r√©seau de neurones, il est n√©cessaire de d√©finir une fonction de perte (loss). Dans notre cas, nous avons deux classes et nous souhaitons maximiser la marge entre les exemples appartenant √† des classes diff√©rentes. Contrairement √† la perte de log-vraisemblance n√©gative utilis√©e pr√©c√©demment, nous allons chercher √† maximiser cet √©cart, ce qui rendra notre m√©thode plus robuste aux nouveaux √©l√©ments.

On utilise le loss max-margin (marge maximale) qui est d√©fini comme :
\(\text{loss} = \max(0, 1 - y_i \cdot \text{score}_i)\)

def loss_function(scores,labels):
  # La fonction .relu() prend le maximum entre 0 et la valeur de 1 - yi*scorei
  losses = [(1 - yi*scorei).relu() for yi, scorei in zip(labels, scores)]
  # On divise le loss par le nombre d'√©l√©ments du mini-batch
  data_loss = sum(losses) * (1.0 / len(losses))
  return data_loss

Entra√Ænement du mod√®le#

Maintenant que l‚Äôon poss√©de les √©l√©ments cl√©s de l‚Äôentrainement, il est temps de d√©finir notre boucle d‚Äôentra√Ænement

for k in range(iteration):
    
  # On r√©cup√®re notre mini-batch random
  inputs,labels=get_batch(batch_size=batch_size)

  # On fait appel au mod√®le pour calculer les scores Y
  scores = list(map(model, inputs))
  
  # On calcule le loss
  loss=loss_function(scores,labels)


  accuracy = [(label > 0) == (scorei.data > 0) for label, scorei in zip(labels, scores)]
  accuracy=sum(accuracy) / len(accuracy)
  
  # Remise √† z√©ro de valeurs de gradients avant de les calculer
  model.zero_grad() 
  # Calcul des gradients gr√¢ce √† l'autograd de micrograd

  # Mise √† jour des poids avec les gradients calcul√©s (SGD)
  learning_rate = 1.0 - 0.9*k/100 # On diminue le learning rate au fur et √† mesure de l'entra√Ænement
  for p in model.parameters():
      p.data -= learning_rate * p.grad
  
  if k % 1 == 0:
      print(f"step {k} loss {loss.data}, accuracy {accuracy*100}%")
