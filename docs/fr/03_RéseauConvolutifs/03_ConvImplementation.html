Implementation de la couche de convolution#

Dans ce cours, nous allons impl√©menter une couche de convolution 1D. Dans le cours expliquant les couches de convolutions, nous avons parl√© exclusivement de la convolution 2D car c‚Äôest cette couche qui est la plus utilis√©e (de loin). Cependant, pour bien comprendre le code et l‚Äôing√©niosit√© des couches de convolution, il est essentiel de commencer par la convolution 1D.

Convolution 1D, comment √ßa marche ?#

De mani√®re assez √©vidente, la convolution 1D est similaire √† la convolution 2D mais sur une seule dimension. C‚Äôest lors de l‚Äôapplication de la couche de convolution 1D que l‚Äôon voit tr√®s rapidement pourquoi une couche de convolution est en fait une boucle for.

Figure extraite du forum.

Les param√®tres classiques de la convolution 2D s‚Äôapplique √©galement :

  • Le padding va ajouter des valeurs au bord du vecteur 1D (d√©but et fin)

  • Le stride d√©finit le pas

  • La kernel_size d√©finit la taille du filtre

  • Les in_channels et out_channels qui correspondent aux nombres de channels d‚Äôentr√©e et de sortie.

  • etc ‚Ä¶

Impl√©mentation#

Passons maintenant √† l‚Äôimpl√©mentation de la couche de convolution 1D.

Note : Sur pytorch, les couches de convolutions ne sont pas impl√©ment√©es en python mais en C++ pour une plus grande rapidit√© de calcul.

import torch
import torch.nn as nn
import torch.nn.functional as F

Dans une couche de convolution, on a une dimension de filtre kernel_size et un nombre de filtre out_channels. L‚Äôid√©e va √™tre de boucler sur l‚Äôensemble de notre vecteur et de calculer les valeurs de sortie en appliquant chaque filtre √† chaque position possible du vecteur d‚Äôentr√©e. A chaque position, on va appliquer une couche fully connected prenant en entr√©e les √©l√©ments contenus dans le filtre de taille \((KernelSize \times InChannels)\) et renvoyant \(OutChannels\) √©l√©ments. C‚Äôest comme si on appliquait une boucle for sur chaque position de notre s√©quence d‚Äôentr√©e.

Note : Sur la figure explicative, il n‚Äôy a qu‚Äôune seule dimension de channel mais en r√©alit√© il y en a plusieurs la plupart du temps.

in_channels = 3
out_channels = 16
kernel_size = 3
kernel=nn.Linear(in_channels*kernel_size, out_channels)

Maintenant, il faut appliquer cette couche de convolution sur l‚Äôensemble des √©l√©ments de notre s√©quence avec un pas stride. On peut √©galement ajouter du padding si on veut que la s√©quence d‚Äôentr√©e soit de la m√™me taille que la s√©quence de sortie.

# Imaginons une s√©quence de 100 √©l√©ments, avec 3 canaux et un batch de 8
dummy_input = torch.randn(8, in_channels, 100)
print("Dimension de l'entr√©e: ",dummy_input.shape)
stride=1
padding=1
outs=[]

# On pad les deux c√¥t√©s de l'entr√©e pour √©viter les probl√®mes de dimensions

Comme pour les convolutions 2D, on peut choisir de diminuer la taille de la s√©quence (ou des feature map si on parle d‚Äôune conv 2D). Pour cela, on peut utiliser un stride sup√©rieur √† 1 ou bien une couche de pooling.

En pratique, l‚Äôutilisation du stride est souvent pr√©f√©r√©e mais nous allons impl√©menter le max pooling pour bien comprendre son fonctionnement.

pooling=2
outs2=[]

  chunk=outs[:,:,i-pooling:i]
  # On prend le max sur la dimension 2, pour le average pooling on aurait utilis√© torch.mean

Maintenant que nous avons compris comment la convolution 1D et le maxpooling fonctionnent, on va cr√©er des classes pour faciliter leurs utilisations.

class Conv1D(nn.Module):
  def __init__(self, in_channels, out_channels, stride, kernel_size, padding):
    super(Conv1D, self).__init__()

    self.in_channels = in_channels
    self.out_channels = out_channels
    self.stride = stride
    self.kernel_width = kernel_size
    self.kernel = nn.Linear(kernel_size * in_channels, out_channels)
    self.padding=padding

  def forward(self, x):
    x=F.pad(x, (self.padding, self.padding))
    # Boucle en une seule ligne pour un code plus concis
    l = [self.kernel(x[:, :, i - self.kernel_width: i].reshape(x.shape[0], self.in_channels * self.kernel_width)) for i in range(self.kernel_width, x.shape[2]+1, self.stride)]
    return torch.stack(l, dim=2)


class MaxPool1D(nn.Module):
  def __init__(self, pooling):
    super(MaxPool1D, self).__init__()
    self.pooling = pooling
    
  def forward(self, x):
    # Boucle en une seule ligne pour un code plus concis
    l = [torch.max(x[:, :, i - self.pooling: i], dim=2)[0] for i in range(self.pooling, x.shape[2]+1, self.pooling)]
    return torch.stack(l, dim=2)

class cnn1d(nn.Module):

    super().__init__(*args, **kwargs)

    return output

class Conv2D(nn.Module):
  def __init__(self, in_channels, out_channels, stride, kernel_size, padding):
    super(Conv2D, self).__init__()

    self.in_channels = in_channels
    self.out_channels = out_channels
    self.stride = stride
    self.kernel_width = kernel_size
    self.kernel = nn.Linear(in_channels*kernel_size**2 , out_channels)
    self.padding=padding

  def forward(self, x):
    b, c, h, w = x.shape
    x=F.pad(x, (self.padding, self.padding,self.padding,self.padding))
    # Sur une seule ligne, c'est absolument illisible, on garde la boucle
    l=[]
    for i in range(self.kernel_width, x.shape[2]+1, self.stride):
      for j in range(self.kernel_width, x.shape[3]+1, self.stride):
        chunk=self.kernel(x[:,:,i-self.kernel_width:i,j-self.kernel_width:j].reshape(x.shape[0],-1))
        l.append(chunk)
    # La version en une ligne, pour les curieux
    #l = [self.kernel(x[:, :, i - self.kernel_width: i, j - self.kernel_width: j].reshape(x.shape[0], ,-1)) for i in range(self.kernel_width, x.shape[2]+1, self.stride) for j in range(self.kernel_width, x.shape[3]+1, self.stride)]
    outs=torch.stack(l, dim=2)
    return outs.reshape(b,self.out_channels,h//self.stride, w//self.stride)
