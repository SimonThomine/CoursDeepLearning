    Application segmentation¬∂

    Qu'est ce que la segmentation ?¬∂

    Dans les notebooks pr√©c√©dents, nous avons vu uniquement des probl√®mes de classification c'est-√†-dire pr√©dire une classe √† partir d'une image (c'est une image de 5, c'est une image de chat etc...).
    Cependant, il existe deux autres cat√©gories d'analyse d'images : la d√©tection et la segmentation.

    D√©tection d'objets dans une image¬∂

    La d√©tection d'objet dans une image consiste √† encadrer un objet dans une image. Par exemple, si je cherche √† d√©tecter les chats et que j'ai un chien et un chat sur l'image, je vais chercher √† entra√Æner un mod√®le qui va dessiner une bo√Æte autour du chat sur l'image.

    Voici une image pour visualiser l'id√©e de ce qu'est la d√©tection d'objet :

    Image provenant du blogpost.

    Comme vous pouvez l'imaginer, c'est une t√¢che tr√®s int√©ressante pour le domaine du traitement d'images mais ce n'est pas ce que nous allons aborder dans ce notebook.

    Segmentation d'objets dans une image¬∂

    La segmentation a pour but de trouver la classe d'appartenance de chaque pixel de l'image (le pixel en position (120,300) appartient √† un chien par exemple). L'id√©e √©tant d'obtenir des informations tr√®s pr√©cises sur le contenu de l'image √† l'√©chelle du pixel.

    Image provenant du blogpost.

    Il existe plusieurs types de segmentation :

    • La segmentation s√©mantique : Segmentation qui consiste √† classifier chaque pixel de l'image ind√©pendamment de l'instance (Toutes les voitures seront colori√©es en rouge par exemple). C'est le type de segmentation que vous voyez sur l'image au dessus.
    • La segmentation d'instance : Cette fois-ci, on va segmenter chaque instance d'une m√™me classe avec une couleur diff√©rente (La premi√®re voiture en rouge, la seconde en orange etc ...).
    • Il existe √©galement d'autres types de segmentation. Pour plus d'informations, vous pouvez consulter le blogpost.

    Dans ce notebook, nous allons faire un exemple de segmentation s√©mantique.

    Architecture du mod√®le : U-Net¬∂

    Contrairement √† un probl√®me de classification d'images, la sortie de notre r√©seau de neurones doit √™tre de la m√™me dimension que l'entr√©e en r√©solution et avoir un channel par classe que l'on d√©sire segmenter (Si l'on veut segmenter dix classes sur des images d'entr√©e de taille $224 \times 224 \times \ 3$ on aura une sortie de taille $224 \times 224 \times \ 10$).
    Une architecture classique de r√©seau convolutifs o√π la r√©solution diminue au fur et √† mesure des couches jusqu'√† une couche fully connected de classification n'est donc pas la bonne m√©thode.
    A la place, on peut utiliser l'architecture U-Net.

    Voici √† quoi ressemble l'architecture U-Net :

    Figure extraite de ce blogpost

    Comme vous pouvez le constaster, l'architecture est plut√¥t atypique. Elle consiste en un chemin de contraction (appel√© encodeur) qui r√©duit la taille de l'image comme un CNN classique et un chemin d'expansion (appel√© d√©codeur) qui augmenter la taille de l'image jusqu'√† une r√©solution identique √† l'entr√©e. Les connexions indiqu√©es par les fl√®ches noires permettent de conserver une information locale pr√©cise tandis que la partie encodeur/d√©codeur permet de d√©tecter des relations plus abstraites entre diff√©rents pixels.
    Cette architecture a √©t√© introduite en premier pour la segmentation d'images m√©dicales dans cet article. Elle est maintenant utilis√©e dans beaucoup de domaines (segmentation, denoising, diffusion etc ...).

    Point sur la convolution transpos√©e¬∂

    Nous avons vu en d√©tail les couches de convolutions qui permettent d'effectuer des op√©rations sur les images avec des param√®tres entra√Ænables. On a vu que, gr√¢ce au param√®tre stride, les couches de convolution nous permettent de conserver la r√©solution initiale ou bien de diminuer cette r√©solution.
    Cependant, on a parfois besoin d'augmenter la r√©solution de l'image (pour la partie d√©codeur du U-Net d√©crit au dessus par exemple).

    Pour cela, on peut envisager plusieurs options :

    • Interpolation suivi d'une convolution : A l'image du pooling mais dans l'"autre sens", cette id√©e consiste √† utiliser une fonction d'interpolation non entra√Ænable pour agrandir artificiellement l'image.
    • Convolution tranpos√©e : Cette op√©ration joue le r√¥le de l'inverse d'une convolution et permet d'agrandir la taille de l'image. Voici un exemple de convolution transpos√©e avec un kernel de taille $2 \times 2$. C'est une op√©ration qui contient des param√®tres entra√Ænables.

    Figure extraite de blogpost.

    Pour ceux qui veulent entrer dans le d√©tail et savoir les diff√©rences principales entre les deux techniques, je vous conseille la lecture de cet article.

    Dans notre impl√©mentation, nous allons utiliser la convolution transpos√©e.

def conv_relu_bn(input_channels, output_channels, kernel_size, stride, padding):
  return nn.Sequential(
    nn.Conv2d(input_channels, output_channels, kernel_size, stride, padding),
    nn.ReLU(),
    nn.BatchNorm2d(output_channels,momentum=0.01)
  )
    
# Idem mais avec la convolution transpos√©e
def convT_relu_bn(input_channels, output_channels, kernel_size, stride, padding):
  return nn.Sequential(
    nn.ConvTranspose2d(input_channels, output_channels, kernel_size, stride, padding),
    nn.ReLU(),
    nn.BatchNorm2d(output_channels,momentum=0.01)
  )

class UNet(nn.Module):
  def __init__(self, num_classes=3):
    super().__init__()
    # encoder
    self.conv1=conv_relu_bn(3, 64, 3, 1, 1)
    self.conv2=conv_relu_bn(64, 64, 3, 1, 1)
    self.maxPool1=nn.MaxPool2d(2, 2)
    
    self.conv3=conv_relu_bn(64, 128, 3, 1, 1)
    self.conv4=conv_relu_bn(128, 128, 3, 1, 1)
    self.maxPool2=nn.MaxPool2d(2, 2)
    
    #Module central
    self.conv5=conv_relu_bn(128, 256, 3, 1, 1)
    self.conv6=conv_relu_bn(256, 256, 3, 1, 1)
    
    # D√©codeur
    self.convT1=convT_relu_bn(256, 128, 4, 2, 1)
    self.conv7=conv_relu_bn(256, 128, 3, 1, 1)
    
    self.convT2=convT_relu_bn(128, 64, 4, 2, 1)
    self.conv8=conv_relu_bn(128, 64, 3, 1, 1)
    
    # On va pr√©dire un channel par classe
    self.conv9=conv_relu_bn(64, num_classes, 3, 1, 1)     
    self.sigmoid=nn.Sigmoid()
  
  def forward(self,x):
    # Encodeur
    x=self.conv1(x)
    x1=self.conv2(x)
    
    x=self.maxPool1(x1)
    
    x=self.conv3(x)
    x2=self.conv4(x)
    x=self.maxPool2(x2)
    
    # Module central
    x=self.conv5(x)
    x=self.conv6(x)

    #D√©codeur
    x=self.convT1(x)
    x=torch.cat((x,x2),dim=1)
    x=self.conv7(x)
    
    x=self.convT2(x)
    x=torch.cat((x,x1),dim=1)
    x=self.conv8(x)
    
    x=self.conv9(x)
    x=self.sigmoid(x)
    return x

    def calculate_class_accuracy(preds, masks, class_idx):
  #On convertit les pr√©dictions en valeur entre 0 et 1 pour chaque classe
  preds = torch.argmax(preds, dim=1)
  
  # On ne r√©cup√®re que les pixels de la classe d'int√™ret
  preds_class = (preds == class_idx).float()
  masks_class = (masks == class_idx).float()
  
  # Calculer la pr√©cision pour la classe choisie
  correct = (preds_class == masks_class).float()
  accuracy = correct.sum() / correct.numel()
  return accuracy
